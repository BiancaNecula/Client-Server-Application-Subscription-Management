Tema 2 Protocoale de comunicatii

In aceasta tema am implementat o aplicatie client-server utilizand protocoalele TCP si UDP. Aplicatia gestioneaza mesajele primite de la clienti TCP (implementat de mine in subscriber.c) si clienti UDP (deja implementati) prin intermediul unui server.
Clientii TCP se pot conecta si reconecta (cea de a doua implicand salvarea mesajelor) la server si pot da subscribe/unsubscribe la anumite topicuri. Aceste topicuri sunt date prin intermediul unor mesaje primite de la clientii UDP sub forma unor structuri care contin topicul, tipul de date si continutul. In momentul cand clientul se deconecteaza acesta are optiunea cand da subscribe sa isi salveze mesajele primite de la respectivul topic in timpul cat este deconectat. Astfel, fiecare client este reprezentat printr-o structura de id, lista de topicuri la care a dat subscribe, lista de mesaje pe care le va primi etc. Toate aceste liste (si inclusiv lista de clienti) sunt tinute intr-o structura de date de tip Linked List (implemantarea acesteia este realizata de mine in cadrul laboratorului de SD in anul 1). Protocolul de comunicare intre clientii TCP si server l-am implementat prin niste structuri ajutatoare: client_msg (rol: trimite informatii server-ului despre ce vrea clientul - subscribe, unsubscribe, exit), pack_to_send (rol: trimite informatii clientului legate de pachetele/mesajele de la clientii udp si de unde provin acestea + se verifica daca este mesaj de exit sau nu). Pe langa structuri, la inceputul unei conexiuni cu un nou client se trimite si id-ul clientului printr-un string.
Tema nu este momentan complet functionala prin intermediul checkerului dar inca lucrez la anumite detalii.
